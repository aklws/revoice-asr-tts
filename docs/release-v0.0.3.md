# Revoice ASR-TTS v0.0.3 更新公告

发布日期：2026-07-10

## 本次更新重点

- 优化麦克风与文本模式切换：切回文本模式时会主动停止麦克风任务，并卸载 `ASR` 与 `emotion2vec+`，避免录音设备和识别链路继续残留工作。
- 调整麦克风 ASR 策略：去掉过碎的短静音切段，改为更偏整段的识别方式，减少重复识别和旧句反复补全的问题。
- 补强静音保护：低能量片段会在送入 ASR 前被裁剪或直接跳过，长时间静默后会重置 ASR 上下文，降低沉默时重复吐旧句的概率。
- 优化模型显存占用：关闭 `Accel GPT`，将 `QwenEmotion` 固定到 `CPU`，并把 `campplus_model` 改为按需加载，减少 `IndexTTS-2` 常驻显存压力。
- 增强参考音频缓存：参考音频特征改为多条目真缓存，切回已使用过的参考音频时可直接复用缓存结果。
- 稳定语义前端加载：语义前端改为首次需要时再加载，在当前 `IndexTTS` 生命周期内复用，降低初始占用。
- 精简主界面：整体字体略微缩小，移除部分重复卡片层级和重复状态文案，信息层次更清晰。

## 细节调整

- 文本模式下不允许启用实验性情感识别；如果在麦克风模式开启后切回文本模式，会自动关闭该功能。
- 参考音频卡片中，重复的引擎状态展示已移除，只保留主要状态通知。
- 麦克风采集实现已切换为 `sounddevice.InputStream`，规避 `QAudioSource` 跨线程带来的计时器警告。
- 文本模式结束后会再次兜底卸载 `ASR` 与 `emotion2vec+`，避免后台残留。

## 模型与下载

- 项目内自动下载模型的路径继续统一为 `ModelScope`。
- `BigVGAN` 默认仓库保持为：
  - `nv-community/bigvgan_v2_22khz_80band_256x`
- `campplus` 默认仓库保持为：
  - `iic/speech_campplus_sv_zh-cn_16k-common`
